# PokeFetch

## 🔰 Introduction
PokeFetch is a web application that retrieves and displays a list of Pokémon using the public PokéAPI. This project is built to demonstrate fundamental web development skills, relying exclusively on basic JavaScript (`fetch`), HTML, and CSS. It fulfills the requirements up to Milestone 2 of the college web development project.

## 🚀 Milestone 1: Project Setup
- **Project Idea:** A Pokémon directory that lists various Pokémon.
- **Public API:** PokéAPI (https://pokeapi.co/api/v2/pokemon)
- **Features Planned (Up to Milestone 2):**
  - Fetching Pokémon data asynchronously.
  - Displaying a loading state while data is being retrieved.
  - Rendering the data dynamically onto a responsive grid.
- **Technologies Used:** HTML5, Vanilla CSS3, Vanilla JavaScript (ES6+).

## 🌐 Milestone 2: API Integration
- **API Calls:** Utilizes the built-in `fetch` API to retrieve data from the PokéAPI.
- **Dynamic Display:** Parses the JSON response to dynamically generate DOM elements (cards) for each Pokémon.
- **Loading State:** A visual loading indicator is displayed while the network request is pending.
- **Responsive Design:** The layout uses modern CSS Flexbox/Grid to ensure the application looks great on mobile, tablet, and desktop screens.

## 🛠️ How to Run Locally
1. Clone or download this repository.
2. Open the `index.html` file in any modern web browser.
3. No server or build process is required! The project uses pure HTML, CSS, and JS.
